### 正确的初始状态到底长什么样？

在 Polymarket / Gnosis / Omen 这类 Conditional Tokens 标准的预测市场里，**池子一开始根本不存在“Yes 池子”和“No 池子”这两个独立的东西**。

真实情况是：

1. 第一个流动性提供者（LP）投入 20,000 USDC（比如）  
2. 智能合约**一次性、成对铸造**  
   - 20,000 个 Yes shares  
   - 20,000 个 No shares  
3. 这 40,000 个 shares 全部交给 LP（LP 持有）  
4. 此时**市场合约里本身是 0 Yes + 0 No**（合约里根本没有持有任何 shares！）

→ 这才是真实初始状态。

### 第一次有人买 Yes 的时候发生了什么（真实链上流程）

有人用 1000 USDC 市价买 Yes：

### 有人卖 Yes 的时候（真实链上流程）

### 核心结论（现在彻底严谨了）

1. **市场上流通的所有 Yes shares 和 No shares 永远是成对产生、成对消灭的**  
   → 总量永远几乎相等（差值最多几百，来自费用等微小因素）

2. **合约本身永远不持有任何一方的 shares**  
   → 所有 shares 都在用户或 LP 手里  
   → 这就是为什么“把 Yes 扔回 Yes 池子”根本不可能——因为根本不存在“Yes 池子”这个实体！

3. **所谓的“FPMM 池子”其实只是一个数学公式 + 成对铸造/烧毁机制**  
   → 它不是像 Uniswap 那样有两个独立的 token 储备池  
   → 而是一个“虚拟的、强制对称”的机制
